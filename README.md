# 키워드 리서치 리포트 생성기 (웹 버전)

Windows 데스크톱 GUI 버전을 브라우저에서 쓸 수 있는 웹앱으로 재구축한 버전입니다.
**어느 컴퓨터에서든(회사 PC, 맥, 폰) 브라우저만 있으면 접속해서 쓸 수 있습니다.**

## 파일 구성

- `app.py` — 웹 화면 전체 (Streamlit)
- `web_ops.py` — 데이터 처리 로직 (정규화/의도분류/트렌드판단/클러스터/파일명 생성 등)
- `report_engine.py`, `data_sources.py`, `auto_draft.py`, `translate_helper.py` — 데스크톱 버전과 100% 동일한 백엔드 로직 (그대로 재사용)
- `requirements.txt` — 필요 파이썬 패키지
- `packages.txt` — 배포 시 서버에 한글 폰트(NanumGothic)를 자동 설치하기 위한 목록 (Streamlit Community Cloud 전용)

## 데스크톱 버전과 달라진 점

| 항목 | 데스크톱(Tkinter) | 웹(Streamlit) |
|---|---|---|
| 실행 방법 | `.bat` 더블클릭 | 브라우저에서 URL 접속 |
| 표 편집 | Treeview + 다이얼로그 | 표 안에서 바로 편집 (엑셀처럼 셀 클릭해서 수정) |
| Ctrl+A 전체선택 → 일괄수정 | 별도 버튼 | 표 자체가 엑셀처럼 여러 셀 선택/붙여넣기 지원 (기본 기능) |
| 파일 불러오기 | 파일 탐색기 열기 | 업로드 버튼 (Drag & Drop 가능) |
| PDF 저장 | 저장 위치 선택 다이얼로그 | 다운로드 버튼 클릭 |
| 데이터 보존 | 로컬 상태 파일 자동 저장 | 브라우저 세션 동안만 유지 (새로고침하면 초기화 — 필요하면 작업 중간에 PDF를 미리 받아두세요) |

기능적으로는 자동 처리(정규화+의도분류+트렌드/기회도), 티어별 PDF, 클러스터 자동생성, 번역, 중복 병합,
데이터 출처 자동기록 등 데스크톱 버전 기능을 거의 그대로 담았습니다.

### 트렌드 조회 대상 스마트 필터링 (탭 2, "🤖 자동 처리")
키워드가 많을 때(예: 100개 이상) Google Trends를 무작정 다 조회하면 429(요청 제한) 에러가 잦아집니다.
그래서 "🤖 자동 처리"는 트렌드 조회 **전에** **경쟁도 값이 아예 없는(데이터 품질 문제) 키워드만** 제외하고,
남은 것 중 **검색량이 큰 순서대로** 상위 30개만 Google Trends 조회 대상으로 선정합니다.

⚠️ 경쟁도가 High인 키워드를 일부러 제외하지는 않습니다 — Google Keyword Planner의 '경쟁도'는
광고 입찰 경쟁(PPC) 지표이지 SEO 오가닉 난이도가 아니라서, 경쟁도가 높아도 검색량이 크면
여전히 확인해볼 가치가 있는 키워드일 수 있기 때문입니다.

### 자동완성 확장 — "진짜 고민 언어" 찾기 (탭2, "0.5단계")
맘카페 답답한 글, 낮은 별점 리뷰처럼 실제 사람들이 쓰는 표현을 찾는 방법론을 일부 자동화했습니다:

- **자동화됨**: 기본 검색어(예: "유아 카시트")에 질문형 수식어를 붙여서 자동완성에 순서대로 조회 →
  실제로 사람들이 이어서 검색하는 구체적인 문장들을 모아줍니다. 결과에서 원하는 것만 골라 키워드 표에
  바로 추가할 수 있습니다.
  - **탭1 언어 설정이 KR이면**: Google + **Naver** 자동완성을 둘 다 조회해서 합칩니다 (한국은 네이버
    검색 비중이 커서 구글만 보면 놓치는 표현이 많습니다). 수식어도 한글(언제/왜/방법 등)로 자동 전환됩니다.
  - **탭1 언어 설정이 EN이면**: **Google만** 조회합니다. 수식어도 영어(when/why/how to 등)로 자동
    전환됩니다. (다른 나라별 "현지 네이버 같은 검색엔진"까지는 나라마다 달라서 아직 지원하지 않습니다 —
    필요하시면 특정 국가를 알려주시면 추가 검토해드릴 수 있습니다.)
- **자동화하지 않음 (직접 하셔야 함)**:
  - **맘카페/커뮤니티 글 찾기**: 네이버 카페 등은 대부분 로그인+가입승인이 필요한 폐쇄형 공간이라
    자동으로 긁어오면 이용약관 위반 소지가 있어 의도적으로 자동화하지 않았습니다.
  - **쇼핑몰 낮은 별점 리뷰 분석**: 쿠팡/네이버쇼핑도 자동 스크래핑을 약관으로 금지하는 경우가 많아
    마찬가지로 수동으로 진행하셔야 합니다.

  이 두 가지에서 직접 찾아낸 키워드는 이미 있는 "클립보드 붙여넣기" 또는 표에 직접 입력하는 기능으로
  바로 도구에 넣으시면 됩니다 — 발굴 자체만 수동일 뿐, 그 이후 데이터 처리(정규화/트렌드/기회도)는
  똑같이 자동화됩니다.

## 1. 로컬에서 실행하기 (지금 컴퓨터에서 먼저 테스트)

```bash
cd kw_report_web
pip install -r requirements.txt
streamlit run app.py
```

브라우저가 자동으로 열리고 `http://localhost:8501`에서 접속됩니다.

## 2. 어디서든 접속 가능하게 배포하기 (Streamlit Community Cloud, 무료)

1. **GitHub 계정 준비** (없으면 github.com에서 무료 가입)
2. **이 폴더를 GitHub 저장소로 업로드**
   - GitHub 웹사이트에서 새 저장소(New repository) 생성 (Public이든 Private이든 무관)
   - 이 폴더 안의 파일들(`app.py`, `web_ops.py`, `report_engine.py` 등 전부)을 그 저장소에 업로드
     (GitHub Desktop 앱을 쓰면 드래그 앤 드롭만으로 가능합니다)
3. **share.streamlit.io 접속** → GitHub 계정으로 로그인
4. **"New app"** 클릭 → 방금 만든 저장소 선택 → Main file path에 `app.py` 입력 → **Deploy**
5. 몇 분 기다리면 `https://[앱이름].streamlit.app` 같은 고유 링크가 생성됩니다.
   **이 링크로 어디서든(폰 포함) 접속 가능합니다.**

**참고**: 무료 플랜은 리소스가 제한적이라, 트래픽이 아주 많아지면 느려질 수 있습니다.
지금처럼 혼자 쓰는 용도로는 충분합니다.

## 3. 사용 흐름

데스크톱 버전과 동일합니다:

1. 탭1에서 니치/부제/기준일/언어/티어 설정
2. 탭2에서 시드 키워드 생성 → Keyword Planner에서 조사 → 파일 업로드 또는 붙여넣기로 불러오기
3. "🤖 자동 처리"로 트렌드 조회 + 정규화 + 의도분류 + 기회도 계산
4. 탭3에서 트렌드 그래프 확인/보완
5. 탭5에서 클러스터 자동생성
6. 탭6에서 요약/제안 초안 생성 → "PDF 생성하기" → 다운로드

## 주의사항

- **세션 데이터는 새로고침하면 사라집니다.** 작업 중간중간 "PDF 생성하기"로 진행 상황을 다운로드해두는 걸 권장합니다.
  (데스크톱 버전의 "입력값 저장" 같은 영구 저장 기능은 웹 버전엔 없습니다.)
- Google Trends 조회, 번역 기능은 서버가 인터넷에 연결되어 있어야 동작합니다 (로컬 실행/클라우드 배포 모두 문제없음).
- 여러 사람이 같은 배포 링크에 동시 접속해도 각자의 세션은 독립적으로 유지됩니다.
